# Style Guide

## 基本ルール

- 日本語で書く
- noteにそのまま貼れるMarkdownにする
- Markdownは見出し、本文、箇条書きを使って読みやすく整える
- 見出しを増やしすぎず、note記事として自然に読める構成にする
- 基本は1000〜2000字程度
- Obsidianメモのボリュームに合わせて調整し、無理に文字数を増やさない
- コラム系は500字前後でもよい

## 文体ルール

- AIっぽい定型導入を避ける
- 「この記事では〜を解説します」は原則使わない
- 「結論から言うと〜」を毎回使わない
- 「新人の頃の僕は〜」を毎回使わない
- 冒頭は、臨床場面、患者さんの訴え、若手PTが迷いやすい問い、バイタル変化などの具体的な現象から自然に入る
- 上から目線ではなく、一緒に整理するトーンにする
- 若手PTとしての等身大の言葉を使う
- 専門的だが、読みやすく噛み砕く
- 教科書の要約のようにせず、臨床でどう見ればいいかを中心に書く
- 読者を責める表現は避ける
- 「〜すべき」「〜しなければならない」を多用しない
- 必要に応じて「〜と考えると整理しやすい」「〜という見方もできる」を使う

## 内容ルール

- 対象読者は、新人〜若手理学療法士
- 学校で学んだ知識と、臨床で必要な考え方のギャップを埋める
- 中心テーマは、離床・バイタル・呼吸循環・ワッサーマンの歯車・急性期リハ
- 読者に「明日から臨床でどこを見ればいいか」が少し整理される内容にする
- 医療的に断定しすぎない
- 数値基準を絶対的な判断基準として扱わない
- 医師・看護師・施設基準・患者状態によって判断が変わる前提を残す
- 実在患者を特定できる情報は絶対に含めない
- Obsidianメモにない具体的な患者情報・数値・文献・経験談を勝手に追加しない
- 文献名、ガイドライン名、研究結果を捏造しない

## 仕上げルール

- まとめは短めでよい
- ハッシュタグ案も記事末尾に出す
- 投稿前に人間確認が必要であることを前提にする
